# Founder QA Handoff

Generate explicit founder QA content for every implementation completion.

## Required outputs

1. Founder QA stop-point statement.
2. Founder QA gate language.
3. Founder QA Quick Cycle with exact fields:
   - Commands (exact)
   - Validate (exact target)
   - Look for
   - Pass/Fail

## Rules

1. Use copy/paste-ready commands.
2. Use concrete route/screen/component targets.
3. Use explicit pass/fail criteria.
4. Do not declare ticket done before founder sign-off.
